### Method 2: Manual Installation (The Old-School Way)
If you want to install an older 3.3.5 WotLK addon or a community-backported mod from the Ascension Discord, you will need to do it manually.
1. Right-click your game launcher shortcut and select **Open file location**.

3. Download your desired addon (ensure it is compatible with client version 3.3.5).
4. Extract the zip folder directly into the `Addons` directory.
5. Ensure the extracted folder does not contain a nested subfolder of the same name.

## Troubleshooting and Custom Class Workarounds

When troubleshooting your **conquest of azeroth addons**, keep in mind that custom realms operate on highly modified game engines. On Project Ascension, the custom Hero class is built on top of the default client's **Rogue** class. This underlying architecture can occasionally cause standard 3.3.5 addons to behave strangely.

According to player experience and community reports, standard class-based UI elements may default to Rogue behaviors. For example, totem bars, pet happiness indicators, and specific mana/rage resources might not display correctly out of the box.

### Fixing Totem Bars and Custom Resources
If you are playing a custom class that utilizes totems or unique resources, standard shaman or druid UI frames might hide these bars because the game client flags your character as a Rogue. Community developers frequently share workarounds in the Ascension Discord. A common player-developed fix involves opening the addon's core `.lua` files and changing any reference of the `SHAMAN` or `DRUID` class to `ROGUE` to force the frames to load.

### How to Manually Edit Addon Files for Rogue Mapping (Actionable Guide)
If you are determined to use a legacy 3.3.5 addon that isn't displaying your class-specific bars (like shaman totems), you can often fix this with a simple text edit:
1. Navigate to your `Interface > Addons` folder and locate the specific addon's directory.
2. Search for the main `.lua` files (often named after the addon itself, e.g., `AddonName.lua`).
3. Open the file in a text editor like Notepad++ or VS Code.
4. Use the search function (`Ctrl + F`) to look for strings containing `SHAMAN`, `HUNTER`, or `PALADIN`.
5. Replace those class references with `ROGUE` in sections of the code that define resource bar visibility.
6. Save the file, launch the game, and type `/reload` in your chat box to apply the changes.

This simple trick forces the client to load Shaman-specific or Hunter-specific UI elements even though the game server technically flags your custom hero as a Rogue.

## FAQ Section

### Where is the best place to download conquest of azeroth addons?
The absolute best and safest place to download them is directly through the official Project Ascension Launcher. The launcher features a dedicated "Addons" tab with one-click installation for fully optimized, pre-verified modifications. For more niche or community-made tools, you can explore the `#addons` and `#addons-discussion` channels on the official Ascension Discord, where volunteer developers actively backport and troubleshoot popular mods.

### Can I use standard WotLK 3.3.5 addons as conquest of azeroth addons?
In theory, yes, because the custom client is built on the 3.3.5 Wrath of the Lich King engine. However, in practice, standard addons can suffer from database conflicts or fail to recognize custom class mechanics. If you choose to install a standard 3.3.5 addon, download it from a reputable legacy database and be prepared to manually adjust settings or edit `.lua` files to ensure compatibility.

### Why are my custom class resources not showing up on my unit frames?
Because Ascension's custom classes are built on top of the base Rogue class, standard unit frames often fail to display custom resources like mana, rage, or class-specific power bars. To resolve this, make sure to download the custom *UI Resources* addon directly from the Ascension Launcher, which is specifically designed to overlay and track these custom resources.

### What should I do if an addon is causing my game to crash?
If you experience frequent crashes or interface errors, log out to the character select screen and click the "AddOns" button in the bottom-left corner. From there, you can disable your recently installed mods one by one to isolate the culprit. Additionally, if you are experiencing severe WowError #132 crashes, try disabling the HD Patch or the DXVK mod in your launcher settings, as these high-fidelity visual mods can sometimes conflict with older hardware or specific UI overlays.
